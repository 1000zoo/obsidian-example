# 빈 스코프란
- 스프링 빈이 스프링 컨테이너의 시작과 함께 생성되어서, 스프링 컨테이너가 종료될 때 까지 유지된다.
	- 스프링 빈이 기본적으로 [[싱글톤 패턴]] 스코프로 생성되기 때문이다.
- 스코프는 빈이 존재할 수 있는 범위를 뜻한다.

# 스프링 빈이 지원하는 다양한 스코프
- 싱글톤
	- 기본 스코프
	- 스프링 컨테이너의 시작과 종료까지 유지되는 가장 넓은 범위
- [[프로토타입 스코프]]
	- 프로토타입 빈의 생성과 의존관계 주입에만 관여
	- 더는 스프링이 관리하지 않는다.
	- 매우 짧은 범위의 스코프
- [[웹 스코프]]
	- 스프링 웹 관련 기능이 들어가야 사용할 수 있다.
	- `request`: 웹 요청이 들어오고 나갈때 까지 유지
	- `session`: 웹 세션이 생성되고 종료될 때 까지 유지
	- `application`: 웹의 서블릿 컨텍스와 같은 범위로 유지

# 빈 스코프 지정
- [[@Scope]] 애노테이션으로 지정해준다.
### 자동 등록 시 ([[컴포넌트 스캔]])
```java
@Scope("prototype")
@Component
public class HelloBean {}
```

### 수동 등록 시
```java
@Scope("prototype")
@Bean
PrototypeBean HelloBean() {
	return new HelloBean();
}
```

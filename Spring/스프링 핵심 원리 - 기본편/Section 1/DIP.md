# Dependency Inversion Principle (의존관계 역전 원칙)

- 프로그래머는 추상화에 의존해야지, 구체화에 의존하면 안된다.
	- 의존성 주입은 이 원칙을 따르는 방법 중 하나다.
- 인터페이스에 의존하라는 뜻 (역할에 의존)
- 클라이언트 코드에 구체화된 클래스가 있으면 안된다.
